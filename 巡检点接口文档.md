# 巡检点（Inspection Points）接口文档

本文档描述了机器人巡检点的增、删、改、查及执行导航的相关 Socket 接口。

## 1. 基本消息格式

所有消息遵循以下标准 JSON 格式：

**发送格式：**
```json
{
  "type": "location",
  "topic": "操作类型",
  "msg": { ... 业务数据 ... }
}
```

**接收（回调）格式：**
```json
{
  "op": "location",
  "topic": "操作类型",
  "status": "ok/error",
  "msg": { ... 返回内容 ... }
}
```

---

## 2. 接口列表

### 2.1 获取巡检点列表
*   **Type**: `location`
*   **Topic**: `get_endpoints`
*   **说明**: 获取当前地图上所有已保存的巡检点。
*   **发送内容**:
    ```json
    {
      "type": "location",
      "topic": "get_endpoints"
    }
    ```
*   **接收内容 (msg 字段为数组)**:
    ```json
    {
      "op": "location",
      "topic": "get_endpoints",
      "status": "ok",
      "msg": [
        {
          "id": 1,
          "pointId": "point_1",
          "name": "点位名称",
          "x": 1.25,
          "y": 0.86,
          "yaw": 0.0,
          "broadcastContent": "播报内容",
          "playOnArrival": true,
          "photoOnArrival": false,
          "pointType": 0,         // 0:普通点, 1:迎宾点, 2:充电点
          "status": "可达",
          "isReachable": true,
          "stayTime": 0.0,
          "needRotate": false
        }
      ]
    }
    ```

### 2.2 创建巡检点
*   **Type**: `location`
*   **Topic**: `add_endpoint`
*   **说明**: 在指定坐标创建一个新的巡检点，可配置到达后的动作。
*   **发送内容**:
    ```json
    {
      "type": "location",
      "topic": "add_endpoint",
      "msg": {
        "name": "点位名称",
        "broadcastContent": "到达后的语音播报内容",
        "playOnArrival": true,   // 到达后是否执行播报/动作
        "photoOnArrival": false, // 到达后是否拍照
        "pointType": 0,          // 点位类型
        "position": {
          "x": 1.25,
          "y": 0.86,
          "yaw": 0.0
        }
      }
    }
    ```

### 2.3 更新巡检点
*   **Type**: `location`
*   **Topic**: `update_endpoint`
*   **说明**: 修改已有巡检点的名称、位置或配置。
*   **发送内容**:
    ```json
    {
      "type": "location",
      "topic": "update_endpoint",
      "msg": {
        "pointId": "point_1",    // 必填：要修改的点位ID
        "name": "新名称",
        "broadcastContent": "更新后的播报内容",
        "playOnArrival": true,
        "photoOnArrival": true,
        "pointType": 1,
        "position": {
          "x": 1.30,
          "y": 0.90,
          "yaw": 1.57
        }
      }
    }
    ```

### 2.4 删除巡检点
*   **Type**: `location`
*   **Topic**: `delete_endpoint`
*   **说明**: 根据 ID 删除指定的巡检点。
*   **发送内容**:
    ```json
    {
      "type": "location",
      "topic": "delete_endpoint",
      "msg": {
        "pointId": "point_1"
      }
    }
    ```

### 2.5 执行巡检点导航（前往某点）
*   **Type**: `location`
*   **Topic**: `start_point`
*   **说明**: 控制机器人移动到指定的巡检点。
*   **发送内容**:
    ```json
    {
      "type": "location",
      "topic": "start_point",
      "msg": {
        "pointId": "point_1"
      }
    }
    ```

---

## 3. 错误处理

当 `status` 为 `error` 时，`msg` 字段通常包含错误信息：
```json
{
  "op": "location",
  "topic": "error",
  "status": "error",
  "msg": {
    "errorMessage": "错误描述信息"
  }
}
```

---

## 4. 辅助点位接口 (2dpath)
*注：本项目中部分底层点位操作也可能使用 `2dpath` 类型，主要用于不带业务属性的坐标管理。*

*   **新增**: `type: 2dpath, topic: add_point`
*   **更新**: `type: 2dpath, topic: update_point`
*   **删除**: `type: 2dpath, topic: delete_point`
*   **查询**: `type: 2dpath, topic: query_point`
