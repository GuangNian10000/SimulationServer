# 路线管理接口文档

本文档基于 `RouteManageFragment` 中的实际代码调用情况整理，涵盖了路线的增删改查、执行以及状态监听。

## 1. 基础消息结构
所有接口遵循 `BaseMessage` 结构：
- `op`: 对应业务类型
- `topic`: 对应具体操作
- `msg`: 具体的参数内容（JSONObject 或 JSONArray）

---

## 2. 路线列表查询 (Query Routes)
查询所有已定义的路线。

- **Op (Type)**: `2dpath`
- **Topic**: `get_routes`
- **请求参数**: `null`
- **响应结构 (`msg`)**:
  ```json
  [
    {
      "id": 1,
      "routeId": "route_x1y2z3...",
      "name": "路线名称",
      "description": "路线描述",
      "pointIds": "point_a1,point_b2,point_c3", // 以逗号分隔的点位ID字符串
      "pointCount": 3,
      "isActive": true,
      "createTime": 1234567890123,
      "updateTime": 1234567890123,
      "loopCount": 1 // 执行循环次数
    }
  ]
  ```

---

## 3. 新增路线 (Add Route)
创建一个新的巡检路线。

- **Op (Type)**: `2dpath`
- **Topic**: `add_route`
- **请求参数 (`msg`)**:
  ```json
  {
    "name": "路线名称",
    "description": "路线描述",
    "isActive": true,
    "points": ["point_id1", "point_id2"], // 点位ID字符串数组
    "loopCount": 1
  }
  ```
- **回调响应**: 成功返回 `status: "success"`。

---

## 4. 更新路线 (Update Route)
修改现有路线的信息或点位。

- **Op (Type)**: `2dpath`
- **Topic**: `update_route`
- **请求参数 (`msg`)**:
  ```json
  {
    "routeId": "route_x1y2z3...",
    "name": "路线名称",
    "description": "路线描述",
    "isActive": true,
    "points": ["point_id1", "point_id2"],
    "loopCount": 1
  }
  ```
- **回调响应**: 成功返回 `status: "success"`。

---

## 5. 删除路线 (Delete Route)
根据 `routeId` 删除指定路线。

- **Op (Type)**: `2dpath`
- **Topic**: `delete_route`
- **请求参数 (`msg`)**:
  ```json
  {
    "routeId": "route_x1y2z3..."
  }
  ```
- **回调响应**: 成功返回 `status: "success"`。

---

## 6. 执行路线巡检 (Start Route Navigation)
触发机器人开始按指定路线进行巡检。

- **Op (Type)**: `navigation`
- **Topic**: `start_route`
- **请求参数 (`msg`)**:
  ```json
  {
    "routeId": "route_x1y2z3..."
  }
  ```
- **回调响应**: 成功启动后返回 `status: "success"`。

---

## 7. 查询路线执行状态 (Get Navigation Status)
主动查询当前机器人的导航/巡检状态。

- **Op (Type)**: `navigation`
- **Topic**: `get_status`
- **请求参数**: `null`
- **响应结构 (`msg`)**:
  ```json
  {
    "state": "COMPLETED", // 状态值：IDLE, RUNNING, COMPLETED, 等
    "routeId": "route_x1y2z3..."
  }
  ```

---

## 8. 路线状态变化通知 (Push Notification)
通过 Socket 监听收到的异步状态通知。

- **Op (Type)**: `navigation`
- **消息结构 (`msg`)**:
  ```json
  {
    "state": "COMPLETED",
    "routeId": "route_x1y2z3..."
  }
  ```
- **业务逻辑提示**:
  - 当 `state` 为 `COMPLETED` 时，表示路线执行完成。
  - 当 `state` 为 `IDLE` 且本地记录有 `currentExecutingRouteId` 时，也视为当前路线执行结束。
